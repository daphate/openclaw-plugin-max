# Безопасность (openclaw-plugin-max)

Проверка плагина по требованиям информационной безопасности.

## Соответствие критериям OpenClaw

- **Расположение:** плагин устанавливается в каталог **extensions**; точка входа в `package.json` задаётся как `./src/index.js` — путь относительный, не выходит за пределы каталога плагина (нет path traversal).
- **plugins.allow:** для работы плагина в конфиге должен быть указан `plugins.allow`, включающий `openclaw-plugin-max`; без явного разрешения плагин не загружается.
- **Статический анализ (code safety):** в исходниках плагина отсутствуют:
  - вызовы `eval`, `new Function`, `child_process` (exec/spawn);
  - подозрительные паттерны (майнинг, обфускация, массовое чтение env + отправка в сеть).
- **Секреты:** токен бота берётся только из конфига (`channels.max.token` / `botToken`); в логи не пишется и не передаётся в сообщения пользователям.

## Реализованные меры

| Требование | Реализация |
|------------|------------|
| Аутентификация webhook | Маршрут `/plugins/max/webhook` зарегистрирован с `auth: "plugin"` — проверка прав выполняется OpenClaw. |
| Ограничение входящих данных | Тело webhook ограничено 512 KB; при превышении возвращается 413 Payload too large. |
| Валидация входных данных | `toId`/`fromId` приводятся к строке/числу; для отправки используется `parseInt(..., 10)`, при NaN отправка не выполняется. |
| Контроль доступа | Доставка сообщений только после проверки `enforceMaxAccess` (dmPolicy, allowFrom, groupPolicy, pairing). |
| Нет выполнения кода извне | Только импорт официального пакета `@maxhub/max-bot-api`, без `eval`/`Function`/`require` произвольных строк. |
| Медиа по URL | При отправке медиа по URL выполняется fetch; лимит 50 MB; при ошибке — fallback на текст. Рекомендуется использовать доверенные URL (агент, внутренние сервисы). |

## Зависимости

- **@maxhub/max-bot-api** (^0.2.2): единственная зависимость; версия зафиксирована в `package-lock.json`. Рекомендуется периодически обновлять и проверять: `npm audit`.

## Рекомендации оператору

1. Хранить токены и секреты в конфиге с ограниченными правами доступа (или в внешнем хранилище секретов, если поддерживается OpenClaw).
2. В production использовать **plugins.allow** с явным списком разрешённых плагинов.
3. Для webhook использовать HTTPS и при необходимости дополнительную проверку (подпись, IP allowlist) на уровне reverse proxy.
4. Периодически запускать `openclaw security audit --deep` и устранять замечания.

## Аудит

Полную проверку конфигурации и установленных плагинов выполняет OpenClaw:

```bash
openclaw security audit
openclaw security audit --deep   # с проверкой кода плагинов и глубоким зондом
```

При установке плагина из непроверенного источника рекомендуется просмотреть исходный код в `src/index.js` и убедиться в отсутствии сетевых обращений с передачей секретов или выполнения произвольного кода.
